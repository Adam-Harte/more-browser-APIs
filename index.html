More Browser APIs

Data Attributes

Data attributes are attributes you can add to DOM elements to store extra data such as strings, numbers or booleans. You define data atrributes in HTML elements with the 'data-' attribute syntax followed by the name you want to give the attrbiute. Data atrributes can be get or set in JavaScript with the dataset property on HTMl elements followed by the data atrribute name wthout the 'data-'.

Element Box Dimensions

We can tet an elements box dimensions, position and coordinates in JavaScript by using the getBoundingClientRect method. When we call this method on an element we get that elements size(width, height), position (top, left, bottom, right) and coordiantes (x, y). The coordinates system is based on the way browsers render pages so from top to bottom in the y and left to right in the x therefore that top left position is 0,0. The right and bottom positons are a combination of the elemnts width, height and x and y values e.g. width + x for right and height + y for bottom.

Element Sizes and Positions

We can get specific information about an element such as size and position information with offset and client properties. Using offsetTop and offsetLeft properties give us the elements top and left positions based on the browser coordinates. We can use clientTop and clientLeft to get the top and left positions of the content inside the elements e.g. content after borders. There is the offsetWidth and offsetHeight properties to get the whole element size including borders etc. There is also clientWidth and clientHeight to get the elements size excluding borders etc. We can also check the actual broswer windows size using window.innerWidth and window.innerHeight. To get the real available browser window size we can use document.documentElement clientHeight or clientWidth properties. Finally when elements are scrollabel we can check the scroll information with scrollHeight and scrollTop. The scrollHeight property gives us the total size of the element including the hidden height thats scrollable, scrollTop gives us the number by how much the element has been scrolled 0 being no scroll.

Scrolling

We can control element scrolling in JavaScript with the scrollTo, scrollBy and scrollIntoView methods. The scrollTo method scrolls an element to the specified x and y values passed whereas scrollBy relatively adds scroll values x and y to the exisiting scroll. The scrollIntoView method scrolls the parent container of that element so that it is in view. All these mthods optionally take in an options object argument where you can define behaviour for example to be smooth so that the scroll position update animates.

Template Tags

The template html tag allows us to define a reusable chunk/template of html markup. This is great to make resuable sections of html that we can grab and import into nodes in JavaScript. We can use these template tags in JavaScript by grabbing them with a DOM query and then via the content property. To use the content in areas of html we can use the document importNode method to pass the template tag content and get back the returned nodes which we can then append or do whatever kind of addition we want into to our html structure.

Loading Script Dynamically

We can load scripts dynamically with JavaScript by creating a script element assigning it the src and other attributes we want and then appending it to the document head.

Timeouts and Intervals

We have the ability to use timers in JavaScript specifically two types of timers. We can use timeouts and intervals. Timeouts allow us to run a block of code once after the allotted time has passed. Intervals allows us to run the same block of code every time the allotted time has passed. We create these timers with the setTimeout and setInterval methods. Both methods take a callback function which is the code we want to execute with the timer and also an argument for the allotted time in miliseconds. These methods also return an id we can store so that we can then cancel the timers at a later point if we wish. These timers are an example of asynchronous code as the code we want to execute doesnt execute right away and instead gets pushed to the task queue which will wait for the stack to be avaialble before moving that task to the stack.

Location and History

The location and history objects allow us to change the web page and navigate the user. Location allows us to modify the pages history whereas history allows us to interact with that history such as navigating the user back/forward in the pages history.
